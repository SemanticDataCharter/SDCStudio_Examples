# Generated Downloads

This directory contains output packages generated by SDCStudio from the source templates.

## Directory Structure

```
downloads/
├── model_pkgs/          # Data model packages (schema definitions only)
├── Enterprise/          # Enterprise-ready application packages
└── OS/                  # Open source/lightweight application packages
```

## Package Types

### Model Packages (`model_pkgs/`)

Core data model definitions without application scaffolding. Each package contains:

| File | Description |
|------|-------------|
| `*.xsd` | XML Schema Definition - validation schema |
| `*.xml` | XML Instance - example data |
| `*.json` | JSON Instance - example data in JSON format |
| `*.jsonld` | JSON-LD Schema - semantic linked data |
| `*.html` | HTML Documentation - human-readable docs |
| `*.rdf` | RDF Triples - semantic web format |
| `*.shacl` | SHACL Constraints - shape validation |
| `*.gql` | GQL Statements - graph database queries |

**Available Model Packages:**

| Package | Source Template | Domain |
|---------|-----------------|--------|
| `Arrest-Report.zip` | NIEM/example-arrest-report.md | Justice |
| `Incident-Report.zip` | NIEM/example-incident-report.md | Justice |
| `Visa-Application.zip` | NIEM/example-visa-application.md | Immigration |
| `Vessel-Arrival-Report.zip` | NIEM/example-vessel-arrival-report.md | Maritime |
| `StatePopulation.zip` | (Sample data) | Demographics |

### Enterprise Packages (`Enterprise/`)

Full-featured application packages for enterprise deployment. Includes:

- Complete data model package (all files from `model_pkgs/`)
- Database schema and migrations
- API endpoints and documentation
- Authentication and authorization scaffolding
- Deployment configurations
- Admin interface components

**Available Enterprise Packages:**

| Package | Description |
|---------|-------------|
| `arrest_report_sdc_project_Enterprise.zip` | Complete arrest reporting system |
| `incident_report_sdc_project_Enterprise.zip` | Incident management application |
| `visa_application_sdc_project_Enterprise.zip` | Visa processing system |
| `vessel_arrival_report_sdc_project_Enterprise.zip` | Maritime arrival reporting |
| `statepopulation_sdc_project_Enterprise.zip` | Population data management |

### Open Source Packages (`OS/`)

Lightweight application packages for quick deployment and customization. Streamlined versions without enterprise features.

## Quick Start: Deploying SDC Apps

SDC apps are **standalone Django applications** that run in Docker. You deploy ONE app first, then import additional apps into it.

### Step 1: Deploy Your First App

1. **Download an Enterprise package** from `Enterprise/` directory
2. **Extract the ZIP** file:
   ```bash
   unzip Enterprise/arrest_report_sdc_project_Enterprise.zip -d my-sdc-project
   cd my-sdc-project/arrest_report_sdc_project
   ```
3. **Follow the generated README.md** inside the extracted folder for complete setup instructions:
   ```bash
   # Copy environment file
   cp .env.example .env

   # Start the application
   docker compose up -d --build
   ```
4. **Access your app** at http://localhost:8000 (default credentials: admin/admin123)

### Step 2: Add More Data Models

Once your first app is running, you can add additional data models using the built-in installer:

**Via Django Admin (Recommended):**
1. Access Django Admin: http://localhost:8000/admin/
2. Navigate to **"SDC4 Tools"** > **"Install App from ZIP"**
3. Upload another Enterprise ZIP file (e.g., `incident_report_sdc_project_Enterprise.zip`)
4. The installer automatically:
   - Extracts and installs the new app
   - Updates Django settings and URLs
   - Runs database migrations
   - Uploads RDF data to GraphDB
5. Restart your server: `docker compose restart web`

**Via Command Line:**
```bash
# Copy the ZIP file into your project, then:
docker compose exec web python manage.py install_app /app/path/to/incident_report_sdc_project_Enterprise.zip
docker compose restart web
```

The home page at http://localhost:8000 automatically lists all installed apps.

### Benefits of Multi-App Projects

- **Single deployment** for related data models
- **Shared SSO authentication** via Keycloak
- **Unified admin interface**
- **Common RDF store** (GraphDB) with OWL 2 reasoning
- **Temporal versioning** for all data via SirixDB

## Using Model Packages (Schema Files Only)

If you only need the schema files without a running application:

```bash
# Extract a model package
unzip model_pkgs/Arrest-Report.zip -d arrest-report-schemas

# View the XSD schema
cat arrest-report-schemas/*.xsd

# View example JSON data
cat arrest-report-schemas/*.json | python -m json.tool
```

### XML Validation (Python)

```python
from lxml import etree

# Load schema
schema = etree.XMLSchema(etree.parse('arrest-report.xsd'))

# Validate document
doc = etree.parse('my-arrest-report.xml')
schema.assertValid(doc)
```

### JSON Schema Validation (JavaScript)

```javascript
const Ajv = require('ajv');
const schema = require('./arrest-report-schema.json');

const ajv = new Ajv();
const validate = ajv.compile(schema);
const valid = validate(myData);
```
